# Design Rule: Responsive Logo

## Principle
A responsive logo adapts to different sizes and contexts while maintaining recognizability at all scales. From favicon (16px) to billboard (meters), each version is optimized for its context.

## Application to Logo

### Simplification Levels
1. **Full version**: Logo with all elements (symbol + name + tagline)
2. **Standard version**: Logo with symbol + name
3. **Compact version**: Symbol/icon only
4. **Micro version**: Ultra-simplified version for favicon/tiny contexts

### Design for Scalability
- Elements must be removable without losing identity
- Each simplification level maintains the visual DNA
- Fine details only in larger versions
- Micro version must work at 16x16 pixels

### Consistency Across Versions
- Same visual language across all versions
- Consistent colors
- Proportions derived from the main version
- Immediate recognizability in any version

### Size Tests
- 16x16: Favicon (micro version)
- 32x32: Small app icon
- 64x64: Avatar/profile
- 128x128: Medium app icon
- 256x256: Large app icon
- 512x512+: General use

### Contextual Adaptation
- Light background version (dark colors)
- Dark background version (light colors)
- Monochrome version
- Version with protection area for complex backgrounds

### Instruction
The logo must be designed with responsiveness in mind. The design must work perfectly from 16px to large sizes. Create a hierarchy of elements that can be progressively removed while maintaining identity. The result must be recognizable at any size.
