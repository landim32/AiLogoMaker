# Regra de Design: Logo Responsivo

## Princípio
Um logo responsivo se adapta a diferentes tamanhos e contextos, mantendo reconhecibilidade em todas as escalas. Do favicon (16px) ao outdoor (metros), cada versão é otimizada para seu contexto.

## Aplicação no Logo

### Níveis de Simplificação
1. **Versão completa**: Logo com todos os elementos (símbolo + nome + tagline)
2. **Versão padrão**: Logo com símbolo + nome
3. **Versão compacta**: Apenas símbolo/ícone
4. **Versão micro**: Versão ultra-simplificada para favicon/tiny contexts

### Design para Escalabilidade
- Elementos devem ser removíveis sem perder identidade
- Cada nível de simplificação mantém o DNA visual
- Detalhes finos apenas nas versões maiores
- Versão micro deve funcionar em 16x16 pixels

### Consistência entre Versões
- Mesma linguagem visual em todas as versões
- Cores consistentes
- Proporções derivadas da versão principal
- Reconhecibilidade imediata em qualquer versão

### Testes de Tamanho
- 16x16: Favicon (versão micro)
- 32x32: App icon pequeno
- 64x64: Avatar/perfil
- 128x128: App icon médio
- 256x256: App icon grande
- 512x512+: Uso geral

### Adaptação Contextual
- Versão para fundo claro (cores escuras)
- Versão para fundo escuro (cores claras)
- Versão monocromática
- Versão com área de proteção para fundos complexos

### Instrução
O logo deve ser projetado pensando em responsividade. O design deve funcionar perfeitamente desde 16px até tamanhos grandes. Crie hierarquia de elementos que possam ser removidos progressivamente mantendo a identidade. O resultado deve ser reconhecível em qualquer tamanho.
